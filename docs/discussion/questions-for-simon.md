# Bob 提给 Simon 的产品决策问题

**创建日期**: 2026-02-02  
**状态**: ✅ Simon 已回复

---

## 背景说明

我们之前设计的系统：
- 学生输入学号 → 系统自动显示他们的班级（section）和老师名字
- 但现在因为隐私问题，我们不保存 section 和 teacher 信息了

新的登录方案：
- 学生使用 **unique ID** 登录（由学号后4位 + 姓名前2位 + 2位随机数字组成）
- 例如：学号 `20241234`，姓名 `张三` → Unique ID 可能是 `3478ZH42`

---

## 问题 1：自动查找班级和老师的功能怎么处理？

**现状**：
- ✅ 旧版本：学生输入学号 → 系统自动显示 "你在 Section A，老师是李老师"
- ❌ 新版本：我们不再保存 section 和 teacher 信息

**技术方案选项**：

| 方案 | 优点 | 缺点 | 技术难度 |
|------|------|------|---------|
| **A. 直接移除功能** | 简单，符合隐私要求 | 学生可能不知道自己的班级信息 | ⭐ 简单 |
| **B. 让学生手动输入** | 学生可以记录自己的信息 | 增加操作步骤，可能输入错误 | ⭐⭐ 中等 |
| **C. 保留但改为可选** | 灵活，学生可选择是否填写 | 需要设计新的界面 | ⭐⭐ 中等 |

**需要 Simon 回答**：
1. 学生**真的需要**在系统中看到自己的班级和老师名字吗？
   - 还是他们已经通过其他方式（如课表、邮件）知道了？
2. 如果需要，学生自己输入这些信息可以接受吗？
3. 这个信息对学生使用系统有多重要？（高/中/低优先级）

**Simon 的回答**：
> 每个学生注册平台时需要输入：学号后4位、姓氏、名字首字母、以及班级编号（Section）。
> 
> 我们可以提前检查是否有学号后4位相同的情况。
> 
> 然后在数据库中保存他们的唯一 ID（在注册时根据学号后4位和姓名首字母生成）。
> 
> 我们不需要向学生显示老师信息，但我们需要知道学生来自哪个班级。 


## 问题 2：老师使用什么方式登录？

**学生的登录方式**（已确定）：
- Unique ID：`3478ZH42`（学号后4位 + 姓名前2位 + 2位随机数字）

**老师的登录方式选项**：

| 方案 | 示例 | 优点 | 缺点 |
|------|------|------|------|
| **A. 类似学生的 Unique ID** | 工号后4位 + 姓名前2位 + 随机数<br>例：`5678LI88` | 统一规则，系统简单 | 老师可能觉得不够专业 |
| **B. 邮箱登录** | `teacher@school.edu` | 专业，老师熟悉 | 需要收集和验证邮箱 |
| **C. 工号登录** | `T20240056` | 清晰明确 | 需要学校提供工号数据 |
| **D. 自定义用户名** | 老师自己设置，如 `li_teacher` | 灵活，易记 | 可能重复，需要检查 |

**需要 Simon 回答**：
1. 老师更习惯用什么方式登录？（考虑他们的技术水平和使用习惯）
2. 我们能获取老师的邮箱或工号吗？
3. 老师和学生使用不同的登录方式，会让系统更清晰吗？

**Simon 的回答**：
> 我们可以使用老师的 HKBU 邮箱手动注册账户，并通过邮件发送密码让他们登录。
> 
> 不支持通过界面修改密码；密码可以从邮件中找回 - 让老师不要删除这封邮件。
> 
> Emma 会提供每个班级的老师姓名和邮箱，以及每个班级的学生姓名和学号。

## 问题 3：既然有了 Unique ID，还需要学生输入学号吗？

**现状**：
- 学生有 Unique ID（如 `3478ZH42`）用于登录
- 但学号（如 `20241234`）可能在其他地方用到

**可能的场景**：

| 场景 | 描述 | 是否需要学号？ |
|------|------|---------------|
| **A. 完全不需要学号** | 学生只用 Unique ID，老师通过姓名识别 | ❌ 不需要 |
| **B. 保留但不显眼** | 学号存在数据库，用于导出或对接学校系统 | ✅ 需要但隐藏 |
| **C. 让学生注册时输入** | 注册时输入学号，系统生成 Unique ID | ✅ 需要 |

**需要 Simon 回答**：
1. 你会用学号做什么吗？比如：
   - [ ] 导出学生成绩时需要显示学号？
   - [ ] 与学校的其他系统对接需要学号？
   - [ ] 老师需要通过学号查找学生？
   - [ ] 没有特殊用途，只是记录
2. 还是说，Unique ID + 姓名就足够了？
3. 如果未来需要对接学校系统，现在不保存学号会很麻烦吗？



**Simon 的回答**：
> 学生不需要输入完整学号，只需要唯一 ID。
> 
> 我们可以通过唯一 ID 识别学生。
> 
> 如果能够匹配学生信息，可能会加快评分速度。
