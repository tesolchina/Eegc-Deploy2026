# Sprint 工作流

## 工作模式

- **Sprint**: 每次集中工作 1 小时
- **每日投入**: 约 2 小时 (2 个 Sprint)

---

## 团队分工

| 角色 | 负责人 | 工具 | 任务 |
|------|--------|------|------|
| 开发 | Kaitai | 自己的 IDE | 写代码，Push 到 GitHub |
| 测试/协调 | Simon | Replit | 每日测试，反馈问题 |

---

## 每日协作流程

```
1. Kaitai 在自己的 IDE 开发
       ↓
2. Kaitai Push 代码到 GitHub
       ↓
3. Simon 在 Replit Pull 最新代码
       ↓
4. Simon 测试功能，记录问题
       ↓
5. 反馈给 Kaitai，下一个 Sprint 修复
```
